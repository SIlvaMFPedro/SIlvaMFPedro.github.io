Features

 Web UI has the following main features:
  • Spectate game
  • Monitoring tournament

    •  The Unity powered game has the following main features:
  • Play chess game

    • Scenarios

     This section will exemplify four possible scenarios of different type of users taking advantage of the Chess Tournament game.


    Join a Tournament


      John, an average consumer has the Chess Tournament Game on his computer. He decides to join a tournament to be able to play, so he selects the tournament in which he wants to play and is added to the pool of players in that tournament.
      Meanwhile, on the Web UI Peter was able to see that John has joined a certain tournament, and all the other players.



    Playing a game


      John, already enrolled in a tournament, now has the ability to play with another player, selected by the system.   If John wins he can play another game with a different opponent selected by the system. This process is repeated until John looses one game or wins the tournament.   Peter, on the Web UI is able to see the moves that John and his opponent are making, and all the other games in the tournament.  



    Watching a game


      Sarah only wishes to see a game. On the game interface she chooses "watch", and selects a tournament, and inside that tournament a specific game. Sarah is able to stop watching anytime and choose another game or tournament to watch.



    Monitoring the tournaments


      On the Web UI Peter can monitor all the tournaments, and their respective games, receive statistics, history of passed games and current state of present games.



    Use cases

     On this section, we expose the most important use cases on the system, according to the frequency of use.


    Unity based Game

    Use case Play a game
    Description The user is able to play the chess game against a system selected opponent using a very simple point and click interface.
    Exceptions If the player looses one game in the tournament, more games will not be possible within that tournament.
    Priority High
    Other information To be able to play any game the player needs to join a tournament

    Use case Join a tournament
    Description Upon opening the Chess Tournament game the player can select one tournament to join to.
    Exceptions The player cannot join a tournament that is already full (has 16 players)
    Priority High
    Other information A tournament does not start until 16 players have joined and the tournament tree is complete

    Use case Watching a game
    Description A user can watch a game taking place from any of the tournaments, through a interface that allows the selection of the tournament first, and then the games taking place within the selected tournament.
    Exceptions If there are no games happening the user cannot watch a game
    Priority Medium
    Other information --



    Web UI


    Use case Watching a game
    Description A user can access the Web UI, select a tournament and a game that has happened or is happening, and access the list of moves made so far in the game.
    Exceptions The user cannot see any games within a tournament if no games have happened in that tournament yet.
    Priority High
    Other information On games happening at the moment of viewing the user can see the moves being made by the players in real time.

    Use case See game and tournament statistics
    Description A user of the Web UI can access statistics of each tournament and game that exists in the platform
    Exceptions --
    Priority Medium
    Other information --
