Data model

In this section basic concepts and relations used in the platform's data model are shown.

 Chess Tourney keeps all the information of the player's current and past games.


 After a tournament is created, a player is able to sign up.


 This player gets an Identification associated to him and after a while he starts playing against another player. During the game, messages with the plays are sent to the Broker to get processed by the aplication. This Messages contain every information that is needed to process the game as it will be shown.


Parameter Type Description
tournament_id int Identification of the tournament
game_id int Identification of the game
id Integer ID of the player
color_pieces string Black/White, color of player pieces
timestamp timestamp Timestamp of the move
from string Position of the piece that the player wants to move
to string Position the player wants the selected piece to move to
