In here you will find all the necessary information, from a developer's point of view, in order to put any of our demonstrations to use in your environment and hopefully helps us out with any problems

General Architecture

The whole architecture involved in our project, involving each of our demonstrations, is the following:

The Framework

As referenceed before, all the work done during this project was in open source engine Unity 5 and we recommend usage of it in order to start working in way that can be free of any problems related to portage of the data.

You can find the free download link in Unity's official website: unity3d.com

All specific assets that we may have used will be referenced in each demonstration's section in detail.

Vertigo

Our Vertigo's demonstration makes use of Kinect camera version 1 as source of information input of the user's position and movement and Oculus Rift DK2/HTC vive to display the gaming environment.

Therefore, in order to tinker with the code, you will have to install the Kinect's Software Development Kit (SDK) which can be found Microsoft's official website: developer.microsoft.com/pt-pt/windows/kinect

Post under Investigation

This demonstration, makes use of Leap Motion sensor as source of information input of the user's hands and fingers positions and movement.

Firstly you will need to install Leap motion's Orion version in order for your computer to detect it, which can be found on Leap's official website: developer.leapmotion.com/orion

Then you will need to install Leap motion's Software Development Kit (SDK) in order to tinker with its code, which can be found on Leap's official website aswell: developer.leapmotion.com/get-started/

This demonstrations architecture can be described by the following image:

The bulk of the work done has been to work on top of Leap's core assets already inserted into UNITY and available at the engine's asset store. The ones in question are the following:

Leap's Unity core assets (developer.leapmotion.com/unity#100): These are related to the caption of hands and fingers positioning abode the Leap's Field of View and are a must to work with the hardware in Unity.

>

Leap's UI Module (available at the same previous download link): This module contains some functions to design canvas in a 3D environment and recognize basic interactions between the user's hands and the UI, like clicking and dragging. This module is NOT a must but we decided to work on top of it, since it presents some easily adapted features to use in our interface design

Lastly, Leap Motion's new version "Orion", saw the discontinuing of gesture based implementations, due to them being considered as not working the way they were originally designed for. Therefore, we have since coded some gesture based scripts that mainly reflect "Swipes" and "Grab" movements. We will soon make this script available in this same section, when we feel those movements are properly optimized for the effect we want them to have in the final solution.

A main prototype of the interface is being worked upon by our team and as soon as it is discussed with the client, will be provided in here for you to have a taste of what the final product we want it to look like.
