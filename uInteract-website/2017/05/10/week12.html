WEEK 12

Posted 10 May 2017

Preparation for real testing environment

Solving issues with calibration






Next week we will finnaly test our demos in an environment with real users at the FCCV so all work done has been to make sure they are in a ready state in order to be tested without breaking issues.More detailed information can be navigated through below.

Vertigo

Graphical part of Vertigo has received improvements when related to last weeks presentations and we started iteration upon some few elements that will improve the immersion of the user when playing the demo.

On another note, an old enemy that we have been battling over the last few weeks has finnaly received due treatment and most efforts done this week were done to making sure he is not there to impact our tests - Oculus Rift connected to the Avatar camera calibration. While we could, and have been, tailoring the camera variables according to our needs at the time, we have never been able to automate the rotation of the camera itself to the Avatar's head when starting the application which resulted in odd rotations that had to be repaired in real time for it to be usable. In a real world testing environment, however, its obvious this is not feasible from an engineering point of view, so we devoted ourselves to solving this issue before the testing date. This resulted in astonishing results, as we were able to identify the root of the problem and solve it in a way that makes sense for the users and won't impact them in any way when playing the demo.

Therefore we are ready to take the demo to the next level and hope to have good reviews of the users tryingn it out.

Avatar

This past week we were able to finnaly identify the root of one of our main issues regarding the movement of the Avatar - Flying Ironman. While the movement itself was coded in the correct way, one of the axis exported from BLENDER into UNITY were inverted which resulted in a change in the translation of the model that was not supposed to be happening. Proper adjustments were made and further testing will be made to understand the impact this solution will have.

Also, we continued configuration of the texture maps to be applied into the Avatar's movements.

Expect some more news regarding this the following weeks.

Dust Under Investigation

Regarding the first module we have made tests with users external to the project but on a inner group scale in order to better realize its current state and what needs to be changed for a 3rd party user.

Work on the second module has continued and the scripts related to the interaction of the user with the testing tree are coming together.

On another note, we have adapted all the code and work done on the following module in order to make it possible to make our presentation with the Leap motion sensor by using our own swipes code and show to the community how it will work in an environment not related to the demo we have been working on since it can be applied to different themes and applications without much effort.

Pong

Following last week's presentation and the feedback we received, we came to the conclusion that we won't have time to deliver a proper demo of our Pong idea. Therefore it will be discarded and won't received further iteration. While this decision doesn't come as a surprise we decided it would be important so that we can shift the effort to the remaining demos that are nearing their final states and we want to fully and extensively test them in order to discover any possible issues that might arise.

That is all for this week. Thanks for reading.
